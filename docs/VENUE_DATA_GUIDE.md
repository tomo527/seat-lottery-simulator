# 会場データ追加ガイド

会場データは`src/data/venues/`に置き、UIコンポーネントへ直接記述しません。型の正本は`src/types/venue.ts`です。

## 型の階層

```text
Venue
└─ layouts: VenueLayout[]
   └─ sections: VenueSection[]
      └─ rows: VenueRow[]
         ├─ seatRanges: { from, to }[]
         └─ excludedSeats?: number[]
```

`Venue`は名称、地域、概算規模、精度、注意書き、情報源を保持します。1会場には公演形式ごとの複数`VenueLayout`を登録できます。`VenueSection.map`は独自SVG簡易図の配置情報です。

## 新しい会場を追加する

1. `src/data/venues/`に会場単位またはデータ群単位の`.ts`ファイルを作ります。
2. URLで使用できる一意かつ安定した`id`を付けます。
3. `region`には定義済み6地域のいずれかを指定します。
4. `notice`にデータの性質、制約、実在データかどうかを明記します。
5. 1つ以上の`layouts`を追加します。
6. `src/data/venues/index.ts`からエクスポートされる配列へ追加します。
7. unit test、UI test、buildを実行し、検索・座席生成・座席図を確認します。

## レイアウトとセクションを追加する

レイアウトは公演形式ごとに分けます。センターステージとエンドステージで座席構成が異なる場合、同じ座席を流用せず別の`VenueLayout`として登録してください。公演によって未確認の変更がある場合は、その制約を`notice`またはレイアウト名で明示します。

セクションには一意の`id`、表示名`label`、簡易図配置`map`、列データ`rows`を指定します。列ラベルは文字列なので、`A`、`AA`、`1`などを表せます。

## 欠番・除外席

連続範囲は`seatRanges`へ複数登録できます。

```ts
{
  label: 'C',
  seatRanges: [{ from: 1, to: 12 }, { from: 15, to: 30 }],
  excludedSeats: [4, 20],
}
```

この例では13・14番は欠番、4・20番は除外席です。いずれも抽選対象になりません。`from`と`to`は1以上の整数で、`from <= to`にします。

## 座席図座標

`map`は`viewBox="0 0 100 76"`を基準に指定します。

- `x`, `y`: 左上の位置
- `width`, `height`: ブロックの幅と高さ
- `shape`: `rectangle`、`rounded`、`arc`
- `rotation`: 中心を基準とする任意の回転角（度）

座標は座席の正確な縮尺ではなく、セクション同士の相対位置を伝える簡易表現です。ラベルが重ならないこと、0〜100付近に収まること、モバイル縮小時にも判別できることを確認してください。

## 情報源と確認日

実在会場の検証済みデータには、確認に使った公開情報を記録します。

```ts
source: {
  title: '情報源ページの名称',
  url: 'https://example.com/source',
  checkedAt: '2026-07-15',
}
```

URLは直接確認できるページ、`checkedAt`は`YYYY-MM-DD`形式にします。情報源の利用条件や著作権にも従ってください。

## データ精度

- `demo`: 動作確認用の完全架空データ。実在会場名に使用しない
- `approximate`: 信頼できる情報を基にしているが、座席単位の完全性までは保証できないデータ。制約を必ず明示する
- `verified`: ユーザー提供の検証済みデータなど、座席範囲と情報源を確認したデータ。確認日を必ず記録する

`verified`は「将来も常に正しい」という意味ではありません。公演レイアウトや改修による差異を説明してください。

## 禁止事項

- 実在会場の座席データを推測して作らない
- ユーザーから検証済みデータが提供されていない実在会場名を使わない
- 実在会場の公式座席図画像をコピー、トレース、埋め込みしない
- 実在チケットサービスの画面、ロゴ、文言、配色を模倣しない
- 情報源・確認日・精度表示を省略して、正確なデータに見せない

公式画像ではなく、権利上問題のない構造化データと`map`座標から本アプリ独自のSVG簡易図を生成してください。
